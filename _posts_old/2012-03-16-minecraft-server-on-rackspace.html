We have minecraft server running 24/7 here at home, which has been running really nicely. We once got caught out by a power failure which corrupted our world (we don't have a UPS) and the world was pretty advanced. Since then, we were running backups of our world and it was working nicely again. However, I was a little perturbed that this machine was on the whole time. So, after playing around with Amazon EC2 and realising that this wasn't for us, I decided to look at rackspace.co.uk - they run small scale servers for a cheap price - £0.02 per hour. You only get billed for the time that the server is running. I'm guessing that this is cheaper than running the server here (as long as we shut the thing down when we're not using it). 

Pros of running the server on rackspace
  • I'm guessing the data is more secure than sat in my house without a machine that is not on a UPS. 
  • Less electricity use
  • other users who are not on our LAN get a better experience (our ADSL upload is slow)

Cons of running on rackspace
  • slight lag compared to running over our LAN
  • time taken to create an image of the server, delete it and then get it running takes time - so incidental use is less likely
  • bandwidth on our ISP is logged, so I will eventually use more bandwidth (I'm not sure how much). 
  • IP address changes every time
