1.5.3 SysVinit

  2. runlevel 1 - single user mode
  3. runlevel 2 - no networking or graphic environment
  4. runlevel 3 - networking
  5. runlevel 5 - networking and graphical environment
  6. runlevel 6 - restart

1.7.3 sequence

  1. first run rc.sysinit script - mounts filesystems and starts LVM
  2. then run rc script - rc/drc[0-9].d directories
  3. run scripts in those directories
  4. Then run rc.local for system specific applications that need to run
  5. start scripts start with an S
  6. kill scripts end start with K
  7. The number following S or K determines the order in which the scripts are invoked.
  8. The script name is the name of the service
  9. Controlling which scripts are run according to each run level involves managing symbolic links
  10. Can manage symlinks manually but also chkconfig will do this consistenyl and easily

1.7.4 Chkconfig someservice

  1. chkconfig –list
  2. sudo chkconfig someservice on
  3. sudo chkconfig someservice off
  4. sudo service someservice [stop|start] does not change current start state

1.7.5 Can write own startup scripts - place a script in /etc/init.d/which has some lines at the top and then use chkconfig –add to enable it or chkconfig –del to disable it

  1. You need to have a chkconfig 2345 10 90 line at the top to let chkconfig know which runlevel it needs to run at and which order it should run in
    1. The 10 and 90 stand for S10 and K90 respectively

1.7.6 Services running

  1. how to find which ones are running sudo service network status
  2. sudo service vsftpd status
  3. Find out service options - sudo service network
  4. All service does is go to the /etc/init.d and run the appropriate script in that directory
  5. sudo service –status-all

1.7.7 Debian derivatives have to have sysvinit-utils chkconfig installed

  1. recent ubuntu versions don't have chkconfig and user update-rc.d utility instead.
    1. sudo invoke-rc.d [status | start | stop] cups
    2. sudo status cups
  2. equivalent of chkconfig would be
    1. sudo update-rc.d cups [defaults | purge]
    2. sudo sysv-rc-conf cups [on|off]

1.7.8 upstart is event driven instead of being a set of serial procedures

  1. event notifications are sent to the init process to tell it to execute certain commands at the time after pre-requisites
  2. being superceded by systemd
  3. config files for upstart are:
    1. /etc/init/rcS.conf
    2. /etc/rc.sysinit
    3. /etc/inittab
    4. /etc/init/rc.conf
    5. /etc/rc[0-5].dz
    6. /etc/init/start-ttys.conf
  4. init runs rcS.conf first which then starts rc.sysinit
  5. rc.sysinit will start LVM filesystems mounting etc and then execute all runlevels in order specified in /etc/inittab
  6. This is accomplished by running rc.conf and passing it the runlevel.
  7. Finally, start-ttys.conf and prefdm.conf (for runlevel 5 only) is run.
  8. initctl options command
    1. start - stop - restart - reload - status - list - emit

1.7.9 systemd

  1. backwards compatible with SysVinit - runlevels targets
  2. telinit program is emulated to work with runlevels
  3. boots faster than other systems
  4. proves aggressive parallelization capabilities
  5. uses socket and d-bus activation for starting services
  6. replaces shell scripts with programs
  7. offers on-demand starting of daemons
  8. keeps tracks of processes using cgroups
  9. supports creating snapshots and restoring of the system state
  10. maintains mount and automount points
  11. implements an elaborate transactional dependency based service control logic
  12. can work as a drop-in replacement for SysVinit
  13. no scripts but instead uses .service files.
  14. Systemd sorts all daemons into their own control groups (cgroups)
  15. uses distro specific config files
    1. e.g. /etc/hostname replaces /etc/HOSTNAME in suse and /etc/sysconfig/network in Red Hat
    2. /etc/vconsole.conf - default keyboard mapping and console font
    3. /etc/sysctl.d/*.conf - drop in directory for kernel sysctl parameters
    4. /etc/os-release - distro id file.
  16. systemctl is the main utility- systemctl [options] command [name]
    1. systemctl list-units -t service –all
    2. systemctl list-units -t service
    3. sudo systemctl start foo
    4. sudo systemctl start foo.service
    5. sudo systemctl start /path/to/foo.service
    6. sudo systemctl stop foo.service
    7. sudo systemctl enable sshd.service
    8. sudo systemctl disable sshd.service
