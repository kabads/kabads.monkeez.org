1 Quotas

Correct method for quotas:

  1. Enable quotas on the filesystem /etc/fstab usrquota grpquota
  2. Remount the file system mount -o remount /home
  3. quotacheck will create a database - quotacheck -cu /home (users only)
  4. Set the quotas with edquota adam to set hard and soft limits
  5. edquota -t will set a grace period (time)
  6. quotaon/quotaoff will enable/disable
  7. quota will report on quota limits
  1. usrquota or grpquota in /etc/fstab -yes
  2. remount mount -o remount /home - yes
  3. quotacheck starts the database - yes quotacheck -cu /home
  4. edquota adam - yes
  5. edquota -t grace period for hard/soft -missed
  6. quotaon or quotaoff - yes
  7. quota to report -yes

  2. remount mount -o remount /home
  3. then quotacheck -cu /home to initialize the database
  4. edquota adam will allow you to set hard and/or soft quota
  5. quotaon or off
  6. quota -t allows you to set a time grace period
  7. quota to report on what is being used - missed

  2. run quotacheck to initialize database and startup

2 LUKS

  1. give the partition to LUKS - cryptsetup luksFormat /dev/VG/Mysecret
  2. make it available with cryptsetup luksOpen /dev/VG/Mysecret SECRET
  3. Format the partition - mkfs.ext4 /dev/mapper/SECRET
  4. mount mount - /dev/mapper/SECRET /mnt/abc
  5. Then close it once finished - cryptsetup -luksClose SECRET
  1. crytpsetup luksFormat /dev/sdaX
  2. cryptsetup luksOpen /dev/sdaX myencfs
  3. crytpsetup -v status myencfs
  4. mkfs.ext4 /dev/mapper/myencfs
  5. mkdir /mountpoint
  6. mount /dev/mapper/myencfs /mountpoint
  7. umount /mountpoint
  8. cryptsetup luksClose myencfs
  1. cryptsetup luksFormat /dev/sda6
  2. cryptsetup luksOpen /dev/sda6 myencfs
  3. mkfs.ext4 /dev/mapper/myencfs
  4. cryptsetup -v status myencfs MISSED
  5. mkdir /mnt/myencfs
  6. mount /dev/mapper/myencfs /mnt/myencfs
  7. umount /mnt/myencfs
  8. cryptsetup luksClose /dev/mapper/myencfs

3 LVM

  1. Create partitions on disk drives (type 8e in fdisk)
  2. Create physical volumes from the partitions
  3. Creat the volume group
  4. Allocate logical volumes from the volume group
  5. Format the logical volumes
  6. Mount the logical volumes (and also update /etc/fstab as needed)

$ sudo pvcreate /dev/sdb1 $ sudo pvcreate /dev/sdc1 $ sudo vgcreate -s 16M vg /dev/sdb1 $ sudo vgextend vg /dev/sdc1 $ sudo lvcreate -L 50G -n mylvm vg $ sudo mkfs -t ext4 /dev/vg/mylvm $ mkdir /mylvm $ sudo mount /dev/vg/mylvm /mylvm Be sure to add the line dev/vg/mylvm /mylvm ext4 defaults 0 0 to /etc/fstab to make this a persistent mount. To grow a logical volume with an ext4 filesystem: $ sudo umount /mylvm $ sudo lvextend -L +500M /dev/vg/mylvm $ sudo resize2fs /dev/vg/mylvm $ sudo mount /dev/vg/mylvm where the plus sign indicates adding space. To shrink the filesystem $ sudo umount /mylvm $ sudo fsck -f /dev/vg/mylvm $ sudo resize2fs /dev/vg/mylvm 200M $ sudo lvreduce -L 200M /dev/vg/mylvm $ sudo mount /dev/vg/mylvm To create a snapshot of an existing logical volume: $ sudo lvcreate -l 128 -s -n mysnap /dev/vg/mylvm To then make a mount point and mount the snapshot: $ mkdir /mysnap $ mount -o ro /dev/vg/mysnap /mysnap To use the snapshot and to remove the snapshot: $ sudo umount /mysnap $ sudo lvremove /dev/vg/mysnap https://www.howtoforge.com/linux_lvm

4 RAID

$ sudo fdisk /dev/sdb $ sudo fdisk /dev/sdc Then set up the array, format it, add to the configuration and mount it: $ sudo mdadm –create /dev/md0 –level=1 –raid-disks=2 /dev/sdbX /dev/sdcX $ sudo mkfs.ext4 /dev/md0 $ sudo bash -c "mdadm –detail –scan >> /etc/mdadm.conf" $ sudo mkdir /myraid $ sudo mount /dev/md0 /myraid Be sure to add a line in /etc/fstab for the mount point /dev/md0 /myraid ext4 defaults 0 0
