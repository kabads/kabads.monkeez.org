1.5 Parameters

1.5.1 usually found in GRUB configuration file

1.5.2 to see what command line a system was booted with type cat /proc/cmdline

1.5.3 There is a long list of parameters

1.5.4 Documentation for parameters can be found at:

  1. in the kernel source in the file Documentation/kernel-parameters.txt
  2. Online at http://kernel.org/doc/Documentation/kernel-parameters.txt
  3. by typing man bootparam
  4. parameters can be specified simply as a value given as an argument or in the form of param=value where the value can be a string, integer, array of integers etc
  5. There should be no intentionally hidden or secret parameters
    1. ro - mounts root device as read-only
    2. root - root filesystem
    3. rd LVM LV - activates the root filesystem in the logical volume specified
    4. rd NO LUKS - disables crypto LUKS detection
    5. rd NO DM - disables DM RAID detection
    6. LANG - is the system language
    7. SYSFONT - is the console font
    8. KEYTABLE - is the keytable filename
    9. rhgb - for graphical boot support on Red Hat systems
  6. The sysctl interface can read and tune kernel parameters at run time.
  7. Current values can be displayed by:
    1. sysctl -a
    2. Each value corresponds to a particular pseudofile residing uner /proc/sys with a directory slashes being replaced by dots
    3. For example, these are equivalent:
      1. sudo sh -c 'echo 1 > /proc/sys/net/ipv4/ipforward'
      2. sudo sysctl net.ipv4.ipforward=1
    4. Browsing through the pseudofiles under /proc/sys will render the same information as sysctl -a
    5. man sysctl will explain details
    6. sudo sysctl -p will load /etc/sysctl.conf
