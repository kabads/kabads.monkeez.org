1.13.6 While modules are loaded, you can always see its status with lsmod

1.13.7 Usually, modprobe is used to load a module

  1. sudo /sbin/modprobe modulename
  2. sudo /sbin/modprobe -r modulename
  3. Modules need to be installed in the proper location - /lib/modules/$(uname -r)/kernel/drivers or etc.
  4. It is impossible to unload a module being used by one or more other modules, which can be found out by lsmod listing.
  5. It is impossible to unload a module that is being used by one more processes, which is also seen by lsmod
    1. However, there are modules which do not keep track of this reference count, such as network device driver modules, as it would make it too difficult to temporarily replace a module without shutting down and restarting the whole network stack.
  6. When a module is loaded with modprobe, the system will automatically load any other modules that need to be loaded first
  7. when a module is unloaded with modprobe -r, the system will automatically unload any other modules being used by the module, if they are not being simultaneously used by any other loaded modules.
  8. /sbin/modinfo mymodule
  9. /sbin/modinfo <pathto>/mymodule.ko
  10. For example, modinfo e1000
    1. displays information about the version, file name which hardware devices the driver module can handle, and what parameters can be supplied on loading
    2. More information can be found in the /sys pseudo-filesystem directory tree;
    3. one would look under /sys/module/e1000 and some, if not all parameters can be read and/or written under /sys/module/e1000/parameters
    4. Many modules can be loaded while specifying parameter values
      1. sudo /sbin/insmod <pathto>/e1000e.ko debug=2 copybreak=256
      2. or for a module already in the proper system location it is easier with:
      3. sudo /sbin/modprobe e1000e debug=2 copybreak=256
    5. Files in /etc/modprobe.d directory control some parameters that come in to play when loading with modprobe
      1. These parameters include module name aliases and automatically supplied options
      2. One can blacklist specific modules to avoid them being loaded.
      3. Settings apply to modules as they are loaded or unloaded, and configurations can be changed as needs change
      4. The format of files in /etc/modprobe.d is simple
        1. One command per line
        2. Blank lines start with a # (comments)
        3. a backslash at the end of a line causes it continue to the next line which makes the file a bit neater.
